# Amazon S3 Data Lake Structure

This project uses Amazon S3 as the primary data lake storage layer.

Bucket name:

healthcare-metrics-hmanthena

The bucket is divided into two major zones.

---

## Raw Layer

The raw layer stores the original CSV files ingested from Google Drive.

Files are partitioned by load_date to support incremental ingestion and historical tracking.

Structure:

s3://healthcare-metrics-hmanthena/raw/

Example:

raw/
 ├── pbj/
 │    └── load_date=2026-03-05/
 │          └── PBJ_Daily_Nurse_Staffing_Q2_2024.csv
 │
 ├── provider/
 │    └── load_date=2026-03-05/
 │          └── NH_ProviderInfo_Oct2024.csv
 │
 └── quality/
      └── load_date=2026-03-05/
            └── NH_QualityMsr_Claims_Oct2024.csv


### Why load_date partitioning?

Partitioning enables:

- incremental ingestion
- historical tracking
- easier reprocessing
- faster query performance

---

## Processed Layer

The processed layer stores transformed datasets generated by AWS Glue ETL.

Data is stored in Parquet format for efficient analytics.

Structure:

s3://healthcare-metrics-hmanthena/processed/

Example:

processed/
 ├── fact_staffing_daily/
 │    └── year_month=2024-04/
 │
 ├── dim_facility/
 │
 └── quality_measures/

Parquet format improves:

- query performance
- compression
- cost efficiency

## Query Layer

The processed datasets are queried using Amazon Athena.

Athena reads the metadata stored in the AWS Glue Data Catalog.

Example tables:

healthcare_processed_db.fact_staffing_daily  
healthcare_processed_db.dim_facility  
healthcare_processed_db.quality_measures  

These tables are used for analytics queries and dashboard visualizations in Amazon QuickSight.
